📖 O que é Git e GitHub?

🔷 Git

Sistema de controle de versão distribuído criado por Linus Torvalds em 2005. Na prática, é um programa que registra todo o histórico de alterações dos seus arquivos, permitindo que você:

  • ✅ Volte para versões anteriores do código
  • ✅ Trabalhe em várias funcionalidades ao mesmo tempo (branches)
  • ✅ Trabalhe offline (tudo está na sua máquina)
  • ✅ Mescle alterações de diferentes desenvolvedores

⚡ Resumo: O Git é o motor que versiona seu código localmente.

🔶 GitHub

Plataforma de hospedagem na nuvem para repositórios Git. Na prática, é onde você armazena e compartilha seus projetos com outras pessoas. Além de guardar seu código, o GitHub oferece:

  • ✅ Repositórios remotos - backup e acesso de qualquer lugar
  • ✅ Colaboração em equipe - várias pessoas podem trabalhar no mesmo projeto
  • ✅ Publicação de sites - GitHub Pages (hospedagem grátis)
  • ✅ Histórico completo - todas as alterações ficam registradas
  • ✅ Compartilhamento - mostre seu portfólio para recrutadores

⚡ Resumo: O GitHub é o servidor onde seus repositórios ficam na nuvem.

🌐 Publique seu projeto no GitHub Pages

▶️ Assista ao vídeo completo no YouTube

O GitHub Pages é um serviço gratuito que hospeda seu site diretamente do repositório. Perfeito para portfólios, projetos pessoais e documentação!

1

Repositório público

Certifique-se que seu repositório está público

2

Acesse Settings → Pages

No repositório, clique em Settings e depois em Pages no menu lateral

3

Escolha a branch

Selecione main, mantenha /(root) e clique em Save

4

Acesse seu site

Após alguns segundos, seu site estará disponível em:
https://seu-usuario.github.io/nome-do-repo/

5

Atualizações automáticas

Sempre que fizer git push, o site é atualizado automaticamente! 🚀

💡 Dica: Para projetos com index.html na raiz, o GitHub Pages já detecta automaticamente!

💻 Enviar para o GitHub via CMD/Terminal

Passo a passo completo para enviar seus projetos do computador para o GitHub usando apenas o terminal (CMD, PowerShell, Bash ou qualquer outro).

1

Navegue até a pasta do projeto

cd caminho/para/sua/pasta

Use o comando cd (change directory) para entrar na pasta do seu projeto

💡 No Windows: clique na barra de endereços da pasta, digite cmd e pressione Enter

2

Inicialize o repositório Git

git init

Isso cria uma pasta oculta .git que versiona seu projeto

3

Configure sua identidade

git config user.name "Seu Nome"
git config user.email "seu-email@exemplo.com"

Define seu nome e e-mail para os commits apenas neste projeto

💡 Use git config --global para definir para todos os projetos

4

Verifique o status dos arquivos

git status

Mostra quais arquivos foram modificados, adicionados ou estão prontos para commit

💡 Os arquivos em vermelho ainda não foram adicionados ao staging

5

Adicione todos os arquivos ao staging

git add .

O . adiciona todos os arquivos da pasta à área de preparação (staging)

💡 Use git add nome-do-arquivo para adicionar apenas um arquivo específico

6

Crie o commit inicial

git commit -m "Commit Inicial"

O -m significa "message" (mensagem). Use mensagens descritivas!

⚠️ Se não configurou nome e e-mail, o Git vai pedir! Volte ao passo 3.

7

Renomeie o branch para main

git branch -M main

O -M renomeia o branch atual (normalmente master) para main

💡 O GitHub usa main como branch padrão atualmente

8

Conecte ao repositório remoto

git remote add origin https://github.com/seu-usuario/nome-do-repo.git

Conecta o repositório local ao repositório remoto no GitHub

⚠️ Substitua a URL pela URL do seu repositório!

9

Envie (push) para o GitHub

git push -u origin main

O -u define o branch padrão (main) e envia os commits para o GitHub

💡 Após o primeiro push, use apenas git push nos próximos envios

📋 Resumo dos comandos

cd pasta Navega até a pasta do projeto
git init Inicializa o repositório Git
git config user.name "Nome" Configura seu nome para commits
git config user.email "email" Configura seu e-mail para commits
git status Verifica o status dos arquivos
git add . Adiciona todos os arquivos ao staging
git commit -m "mensagem" Cria um commit com mensagem
git branch -M main Renomeia o branch para main
git remote add origin URL Conecta ao repositório remoto
git push -u origin main Envia os commits para o GitHub

🔐 Configurações do Git

Comandos essenciais para configurar sua identidade no Git, especialmente em computadores compartilhados como os da escola.

1

Configurar identidade (local)

git config --local user.name "Seu Nome"
git config --local user.email "seu-email@email.com"
git config --local credential.helper manager

Configura nome, e-mail e cache de credenciais apenas para este projeto

✅ Use --local em computadores compartilhados (escola, lan house, etc.)

2

Verificar configurações

git config --local user.name
git config --local user.email

Mostra o nome e e-mail configurados apenas para este projeto

💡 Se não mostrar nada, é porque não está configurado

3

Remover configurações (local)

git config --unset user.name
git config --unset user.email

Remove as configurações apenas deste projeto (local)

⚠️ Use antes de desligar o computador da escola para garantir sua segurança!

4

Listar todas as configurações

git config --list

Mostra todas as configurações do Git (locais + globais)

💡 Use git config --local --list para ver apenas as locais

📋 Resumo dos comandos de configuração

git config --local user.name "Nome" Configura nome local
git config --local user.email "email" Configura e-mail local
git config --local credential.helper manager Configura cache de credenciais local
git config --local user.name Verifica nome local
git config --local user.email Verifica e-mail local
git config --unset user.name Remove nome local
git config --unset user.email Remove e-mail local
git config --list Lista todas as configurações
git config --local --list Lista apenas configurações locais
git config --global --list Lista apenas configurações globais

🛡️ Dica de segurança: Sempre use --local em computadores compartilhados e remova as configurações ao finalizar!
