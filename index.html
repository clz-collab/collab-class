📖 O que é Git e GitHub?

🔷 Git

Sistema de controle de versão distribuído criado por Linus Torvalds em 2005. Na prática, é um programa que registra todo o histórico de alterações dos seus arquivos, permitindo que você:

  • ✅ Volte para versões anteriores do código
  • ✅ Trabalhe em várias funcionalidades ao mesmo tempo (branches)
  • ✅ Trabalhe offline (tudo está na sua máquina)
  • ✅ Mescle alterações de diferentes desenvolvedores

⚡ Resumo: O Git é o motor que versiona seu código localmente.

🔶 GitHub

Plataforma de hospedagem na nuvem para repositórios Git. Na prática, é onde você armazena e compartilha seus projetos com outras pessoas. Além de guardar seu código, o GitHub oferece:

  • ✅ Repositórios remotos - backup e acesso de qualquer lugar
  • ✅ Colaboração em equipe - revisão de código via Pull Requests
  • ✅ Gerenciamento de tarefas - Issues e Projects
  • ✅ Publicação de sites - GitHub Pages (hospedagem grátis)
  • ✅ Automação - GitHub Actions (CI/CD)

⚡ Resumo: O GitHub é o servidor onde seus repositórios ficam na nuvem.

🖥️ Git

  1. Acesse git-scm.com
  2. Baixe a versão para seu sistema
  3. Execute o instalador
  4. Deixe as opções padrão
  5. Verifique: git --version

📁 GitHub Desktop

  1. Acesse desktop.github.com
  2. Baixe e instale
  3. Faça login com sua conta GitHub
  4. Clone ou crie repositórios
  5. Interface gráfica amigável

🌐 GitHub Web

  1. Acesse github.com
  2. Crie uma conta gratuita
  3. Verifique seu e-mail
  4. Crie seu primeiro repositório
  5. Explore a interface web

🌐 Publique seu projeto no GitHub Pages

▶️ Assista ao vídeo completo no YouTube

O GitHub Pages é um serviço gratuito que hospeda seu site diretamente do repositório. Perfeito para portfólios, projetos pessoais e documentação!

1

Repositório público

Certifique-se que seu repositório está público (ou tenha GitHub Pro para privado)

2

Acesse as configurações

No repositório, clique em Settings (Configurações) → Pages no menu lateral

3

Escolha a branch

Em "Branch", selecione main ou master e clique em Save

4

Pasta raiz (root)

Mantenha /(root) como pasta padrão para projetos simples

5

URL do site

Após alguns segundos, seu site estará disponível em:
https://seu-usuario.github.io/nome-do-repo/

6

Atualizações automáticas

Sempre que fizer git push na branch selecionada, o site é atualizado automaticamente! 🚀

💡 Dica: Para projetos com index.html na raiz, o GitHub Pages já detecta automaticamente!

💻 Enviar para o GitHub via CMD/Terminal

Passo a passo completo para enviar seus projetos do computador para o GitHub usando apenas o terminal (CMD, PowerShell, Bash ou qualquer outro).

1

Abra o terminal na pasta do projeto

No Windows: Shift + Botão Direito → "Abrir PowerShell/CMD aqui"

No Mac/Linux: Ctrl + Alt + T e navegue até a pasta com cd caminho/da/pasta

2

Inicialize o repositório Git

git init

Isso cria uma pasta oculta .git que versiona seu projeto

3

Adicione os arquivos ao staging

git add .

O . adiciona todos os arquivos. Use git add nome-do-arquivo para adicionar apenas um

4

Crie o primeiro commit

git commit -m "Primeiro commit"

O -m significa "message" (mensagem). Sempre use mensagens descritivas!

5

Crie um repositório no GitHub Web

Acesse github.com/new, dê um nome e NÃO marque "Add a README"

⚠️ Importante: Deixe o repositório vazio para evitar conflitos!

6

Conecte o repositório local ao remoto

git remote add origin https://github.com/seu-usuario/nome-do-repo.git

Substitua seu-usuario e nome-do-repo pelos seus dados

7

Envie (push) para o GitHub

git push -u origin main

O -u define o branch padrão. Se der erro, tente git push -u origin master (dependendo da versão do Git)

8

Verifique no GitHub Web

Atualize a página do repositório no GitHub. Seus arquivos já estarão lá! 🎉

📋 Comandos Git essenciais

git status Verifica o status dos arquivos
git log Mostra o histórico de commits
git branch Lista os branches existentes
git checkout -b novo-branch Cria e muda para um novo branch
git pull Baixa atualizações do repositório remoto
git clone URL Clona um repositório existente
