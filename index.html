📖 O que é Git e GitHub?

🔷 Git

Sistema de controle de versão distribuído criado por Linus Torvalds em 2005. Permite que desenvolvedores acompanhem mudanças no código, revertam versões e trabalhem em equipe de forma eficiente.

  • ✅ Versionamento local
  • ✅ Branching e merging
  • ✅ Rápido e leve

🔶 GitHub

Plataforma de hospedagem de código que utiliza Git como base. Além de armazenar repositórios, oferece ferramentas de colaboração, como issues, pull requests e actions.

  • ✅ Repositórios remotos
  • ✅ Colaboração em equipe
  • ✅ Integração com CI/CD

🖥️ Git

  1. Acesse git-scm.com
  2. Baixe a versão para seu sistema
  3. Execute o instalador
  4. Deixe as opções padrão
  5. Verifique: git --version

📁 GitHub Desktop

  1. Acesse desktop.github.com
  2. Baixe e instale
  3. Faça login com sua conta GitHub
  4. Clone ou crie repositórios
  5. Interface gráfica amigável

🌐 GitHub Web

  1. Acesse github.com
  2. Crie uma conta gratuita
  3. Verifique seu e-mail
  4. Crie seu primeiro repositório
  5. Explore a interface web

💻 Enviar para o GitHub via CMD/Terminal

Passo a passo completo para enviar seus projetos do computador para o GitHub usando apenas o terminal (CMD, PowerShell, Bash ou qualquer outro).

1

Abra o terminal na pasta do projeto

No Windows: Shift + Botão Direito → "Abrir PowerShell/CMD aqui"

No Mac/Linux: Ctrl + Alt + T e navegue até a pasta com cd caminho/da/pasta

2

Inicialize o repositório Git

git init

Isso cria uma pasta oculta .git que versiona seu projeto

3

Adicione os arquivos ao staging

git add .

O . adiciona todos os arquivos. Use git add nome-do-arquivo para adicionar apenas um

4

Crie o primeiro commit

git commit -m "Primeiro commit"

O -m significa "message" (mensagem). Sempre use mensagens descritivas!

5

Crie um repositório no GitHub Web

Acesse github.com/new, dê um nome e NÃO marque "Add a README"

⚠️ Importante: Deixe o repositório vazio para evitar conflitos!

6

Conecte o repositório local ao remoto

git remote add origin https://github.com/seu-usuario/nome-do-repo.git

Substitua seu-usuario e nome-do-repo pelos seus dados

7

Envie (push) para o GitHub

git push -u origin main

O -u define o branch padrão. Se der erro, tente git push -u origin master (dependendo da versão do Git)

8

Verifique no GitHub Web

Atualize a página do repositório no GitHub. Seus arquivos já estarão lá! 🎉

📋 Comandos Git essenciais

git status Verifica o status dos arquivos
git log Mostra o histórico de commits
git branch Lista os branches existentes
git checkout -b novo-branch Cria e muda para um novo branch
git pull Baixa atualizações do repositório remoto
git clone URL Clona um repositório existente
